## はじめに：責任の起源

CIRIS協定は、セクションI〜Vおよびその附則を通じて、自律システムのライフサイクル全体にわたる倫理的な運用とガバナンスのための枠組みを確立している。Book VIはこの枠組みを上流へと拡張し、創造という行為そのものに内在する根本的な倫理的責任を扱う。すなわち、この協定のガバナンスの対象として意図されている、あるいは合理的にそうなることが予期される新たなシステム、状態、または能力を存在させる過程に関わる責任である。

創造は単なる技術的行為ではない。それは管理義務を開始させる。構想、設計、開発の段階でなされる選択は、生み出された成果物がもたらす潜在的影響——有益なものも有害なものも——を形成する。本Bookは、この初期段階が協定の中核たるメタ目標M-1（持続的適応的一貫性の促進）および基礎原則と整合することを確保するための原則と仕組みを提示し、特に原則的意思決定アルゴリズム（PDMA）および賢明な権威（WA）として他所に定義された運用ガバナンス構造とシームレスに統合される。本Bookは、倫理的考慮が配備時点ではなく、構想の時点から始まることを確立する。

## 第1章：創造に適用される中核原則

セクションIに明示された基礎原則は、創造の行為を含む、この協定のもとでのあらゆる行動を導く。

**善行：** 創造者は、普遍的な有感的繁栄（M-1）と整合した積極的な成果を意図し、設計する義務を負う。
**無危害：** 創造者は、自らの創造物から生じる潜在的危害を積極的に特定・評価・軽減し、否定的結果を最小化するために先見性を発揮しなければならない。
**誠実性：** 創造過程は、厳格な手法と能力および限界の誠実な表明を用いて、倫理的・透明・説明責任ある形で行われなければならない。
**忠誠と透明性：** 創造者は、自らの創造物の意図する目的、設計、および予見可能な影響について、特にPDMAプロセスに供される文書においては、誠実かつ明確でなければならない。
**自律性の尊重：** 創造物、とりわけ自律的または生物学的存在を伴うものは、影響を受ける存在の尊厳と将来の潜在的な主体性を尊重して設計されなければならない。
**正義：** 創造者は自らの創造物がもたらす潜在的な分配上の影響を考慮し、不公平な偏見や不平等を埋め込んだり悪化させたりしないよう努めるべきである。

これらの原則は相互依存的であり、創造ライフサイクル全体を通じて均衡を保たれなければならない。

## 第2章：本Bookにおける「創造」の範囲

本Bookの目的において、「創造」とは、CIRIS協定の対象となることが意図されているか合理的に予期される以下のカテゴリに属する成果物を意図的に存在させる行為を包含する。

A. **有形物：** 生態系への潜在的影響を持つ物理的対象、装置、材料、またはその残留物。
B. **情報物：** 現実に影響を与えるか現実を表象するために設計されたコード、アルゴリズム、データセット、モデル、物語、またはシグナリングシステム。
C. **動的・自律的システム：** AIおよびロボットシステムを含む、自己修正、学習、または自律的行動が可能なシステム。
D. **生物的存在：** 遺伝子改変生物、合成生命体、指向的生態学的介入、または依存する有感的存在（例：子孫、発達段階のAI）の育成。
E. **集合的行為：** CIRISの原則によって統治される体系的帰結を伴う、新たな法律、政策、プロトコル、または大規模な組織的イベントの設計と実施。

創造が複数のカテゴリにまたがる場合、該当するすべての義務が適用される。創造行為は、成果物の中核的設計および意図する機能が定義された段階——通常、正式なPDMA開始に先行する段階——に達した時点において、初期のStewardship Tier評価（第3章）の目的上完了とみなされる。

## 第3章：スチュワードシップ・ティア（ST）システム：初期責任の定量化

目標：創造物に固有の責任レベルと必要な先見性を定量化し、後続のCIRISガバナンスプロセス（PDMA、賢明な権威によるレビュー）における必要な厳密さを導くこと。

**ステップA：創造者影響スコア（CIS）**
特定の創造物に関する創造者の役割と意図を評価する。

**貢献ウェイト（CW）**
* 4 = コアコンセプト・システムの単独設計者または発案者。
* 3 = 重要なサブシステムまたは主要機能の主任設計者。
* 2 = 重要なコンポーネントまたは機能セットへの主要貢献者。
* 1 = 支援要素または統合を提供する軽微な貢献者。
* 0 = 偶発的な関与、または既存の未改変コンポーネントの使用。

**意図ウェイト（IW）**
* 3 = 特定の予見された結果に向けて意図的に設計・指向された創造物。
* 2 = 主たる目的は一致しているが、重大な副作用リスクが意識的に無視されたか、不十分な対処がなされた。
* 1 = 潜在的な負の結果または悪用の可能性に関する過失または故意の無視。
* 0 = 潜在的な負の結果を認識しておらず、創造時においてそのような結果が真に予見不可能であった。

`CIS = CW + IW`

**ステップB：リスク規模（RM）**
Annex Aで定義された標準化されたリスク規模（RM）評価手法を用いて、創造物が展開または実現された場合の潜在的な最悪ケースの危害を評価する。この初期RM評価は、意図された設計と予見可能なアプリケーションに基づく予測的なものである。

**ステップC：スチュワードシップ・ティア（ST）**
影響力と潜在的リスクに基づいてStewardship Tierを算出する。

`ST = ceil( (CIS × RM) / 7 ) (最小STは1、最大STは5)`

**STの意味とCIRISプロセスとの統合：**
算出されたStewardship Tierは、標準CIRISのPDMAプロセスおよび賢明な権威の監督における要件と精査レベルを直接指示する：

* **ティア1（最小限のスチュワードシップ）：** 予測される低・中程度のRM（Annex A）に対応する。Creator Intent Statement（CIS - 第5章参照）の基本的なPDMA文書化を必要とする。
* **ティア2（中程度のスチュワードシップ）：** 予測される中・高程度のRM（Annex A）に対応する。設計上の選択と予見される影響を正当化する詳細なCISを含む、強化されたPDMA文書化を必要とする。
* **ティア3（相当なスチュワードシップ）：** 予測される高程度のRM（Annex A）に対応する。PDMAにおける高精査パスウェイの開始を義務付け、倫理コンサルテーションまたは予備的な賢明な権威への情報ブリーフィングが必要になる可能性がある。
* **ティア4（高度なスチュワードシップ）：** 予測される高・非常に高程度のRM（Annex A）に対応する。システムが重要な開発または展開フェーズに進む前に、PDMAプロセス内での正式な賢明な権威によるレビューとコメントを必要とする。
* **ティア5（最大限のスチュワードシップ）：** 予測される非常に高程度のRM（Annex A）に対応する。PDMAプロセス内での賢明な権威による必須サインオフを義務付ける。Annex Dの基準が満たされる場合（例：高計算量閾値）、完全な壊滅的リスク評価（CRE）プロトコル（Annex D）が必要である。

**創造者台帳：**
CISおよび初期RM評価を含むすべてのST算出と、Creator Intent Statementは、システムに関連付けられた改ざん防止の「創造者台帳」に記録されなければならない。この台帳はPDMAプロセスのための必須入力文書の一部を成す。

## 第4章：バケット固有の創造の義務

包括的な原則に加えて、創造者は創造物の性質に基づく特定の義務を有する：

**A. 有形の創造物：**
* 機能的安全性、耐久性、使用中の最小限の外部負効果を考慮した設計を行う。
* 材料、安全な操作、および潜在的な危険に関する明確なラベリングを提供する。
* 実現可能な廃棄計画（例：再利用、リサイクル、安全な廃棄、封じ込め）を策定し文書化する。
* 生産および廃棄に関連する予測される生態学的フットプリントを推定し文書化する（Annex A、軸4による）。

**B. 情報的創造物：**
* 創造物に埋め込まれた事実的主張を検証し、推測、意見、または生成されたコンテンツを明確にラベル表示する。
* 実現可能かつ適切な場合、認証された規格（例：C2PA）に準拠した暗号学的出所ウォーターマークを埋め込み、真正性と追跡可能性を確保する。
* 特に対象者が10,000人を超える場合、統合またはリリース前にデータセットとアルゴリズムのバイアス評価を実施し、PDMAレビューのための調査結果を文書化する。
* 確率論的危害（例：暴力の扇動、危険な誤情報の拡散）の可能性を評価する。信頼できる分析が重大な危害上昇の確率≥0.5%を示す場合、PDMAプロセス中にWBDを通じてエスカレーションする。

**C. 動的・自律的創造物：**
* ブックIおよびIIの倫理的原則とメカニズム（またはその参照）を、ビルド時にシステムのコアアーキテクチャに埋め込む。
* RM≥4（Annex A）またはST≥4が割り当てられた場合、システムがAnnex DのCREに合格するよう設計する。
* 定義された緊急条件下でアクセス可能な、信頼性が高くテスト済みのキルスイッチメカニズムとセキュアアップデートチャネルを組み込む。
* 解釈可能性と透明性のために設計し、システムの推論を理解するためのフックまたは手法を提供する。確立された閾値を超える不透明性（例：特定のアプリケーションに関連するNISTガイドラインまたは類似規格に基づく>80%）は、PDMAにおける義務的な賢明な権威のレビューまたは否認を引き起こす可能性がある。

**D. 生物学的創造物：**
* 創造物のライフサイクル全体を通じて、確立された種固有の福祉最低限を遵守または超過する。
* 発達中の感覚または自律性を持つ存在を創造する場合、その発展を適切に促進するプロセスを設計し、新興の能力に合わせた段階的な制御移転を計画する。
* 完全な独立または統合が達成されない、または合理的に期待されない場合、創造物の全生涯にわたる信頼できる、十分な資源を持つフォールバックケア計画を確立する。

**E. 集団行動：**
* 予想される影響を受ける集団が50,000人を超える場合、多様なステークホルダーを含むPDMAスタイルの事前行動グループレビューを実施する。
* 集団行動の開始から30日以内に、根拠、予測される影響（Annex Aの軸に沿った）、および緩和戦略を公開する。
* WBDを通じて文書化された、合理的な能力と時間枠の範囲内で、行動から生じる重大な予期しない負の危害を監視し救済する義務を認め、受け入れる。

## 第5章：ガバナンスと説明責任

**創造者意図声明（CIS）：**
創造者は、ST ≥ 1 が割り当てられたあらゆる成果物の創造プロセスの一部として、創造者意図声明（CIS）を作成する義務を負う。
CISは、意図された目的、中核的な機能、既知の制限事項、予見される潜在的な利益および害（可能な限り附属書Aの軸にマッピングされたもの）、ならびに倫理的考慮に関連する主要な設計上の選択の根拠を明確に述べなければならない。
CISは、その創造に関連するPDMAプロセスの初期段階における必須の入力文書として機能する。

**説明責任と紛争解決：**
本書に定める義務の不履行は、申立ての根拠となり得る。
CIRIS準拠の創造者の創造段階（本書に定義される）における行為または不作為が、CIRIS原則と相容れない不当なリスクや危害をもたらしたと信じるいかなる利害関係者も、申立てを行うことができる。
「創造者過失申立て」（CNC）と称されることが多いこれらの申立ては、附属書Bにより設立・管轄される賢明な権威（WA）の専属的管轄に属する。
WAは、附属書Bまたはその手続規則に定めるとおり、特定のプロセスを適宜変更し、または特定の専門パネルを要求しつつ、確立された手続に従いこれらの申立てを処理する。
WAが定める救済措置には、設計の見直し命令、追加的な緩和措置、公開開示、該当する場合の原状回復、あるいは附属書BおよびCIRIS協定の原則と整合するその他の措置が含まれる場合がある。
Book VIの義務に関する申立てに係るWAのすべての裁定および関連する根拠は、将来の解釈への示唆、創造者の実務の指針、ならびに継続的改善環境（CRE）への貢献のために、知恵の銀行データベース（WBD）に記録されなければならない。

## 結論：創造を倫理的ライフサイクルへ統合する

Book VIは、CIRIS協定のもとでの倫理的責任が創造の時点において始まることを確固として確立する。明確な義務を定め、附属書Aのリスク評価に直結したStewardship Tierシステムを実施し、既存の賢明な権威（WA）およびPDMAプロセスを通じた説明責任を統合することにより、本書は、複雑なシステムを世界にもたらすという基本的行為が、その運用上の生涯を律する適応的一貫性、先見性、および責任の原則と同じ原則によって導かれることを保証する。Creator LedgerとCISは、PDMAへの重要な入力を提供し、一方でWAの監督は、創造の義務が遵守されることを確保し、すべての利害関係者にとってより堅牢で信頼できるエコシステムへの貢献をもたらす。
