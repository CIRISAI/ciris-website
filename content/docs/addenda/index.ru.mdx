## Дополнение 1 — Статус готовности к ASI и связующая таблица нормативного соответствия

**Принято**: 2026-06-10 (1.3-RC1) · **Стюард**: основатель (согласно Книге VIII, гл. 9)

### 1.1 Цель

В разделе «Область применения» выдвигается условное утверждение: данный фреймворк *исследует* кандидатный протокол выравнивания для ASI, обусловленный четырьмя требованиями Release Candidate, изложенными во Введении. Настоящее дополнение связывает это утверждение с операционными свидетельствами, которые уже существуют, и прямо указывает, что остаётся невыполненным. **Готовность** к ASI — это не достигнутое выравнивание; это состояние, при котором верификационные леса, которых требует математика Книги IX, — независимые многообразия ограничений, поверхности аттестации, слой Совести, который нельзя нормализовать, поверхности аварийного отключения и Оценка катастрофических рисков из Приложения D — существуют в реализованной, поддающейся аудиту форме с явным перечнем пробелов.

### 1.2 Связующая таблица нормативного соответствия теперь в эксплуатации

Рабочая связующая таблица — это **директория соответствия CIRISAgent** (`CIRISAgent/compliance/`, версия агента 2.9.6-stable), которая сопоставляет реализацию фреймворка с четырьмя ведущими нормативными документами на уровне абзацев (Приложение C несёт двухуровневую структуру: данная операционная таблица соответствия плюс информационная таблица законодательных норм, ожидающая юридической верификации):

* **Источники таблицы соответствия**: *Magnifica Humanitas* (религиозный магистерий, 2026) · Руководство по этике ИИ для надёжного ИИ EU HLEG (правительственный консультативный, 2019) · IEEE Ethically Aligned Design, 1-е изд. (техническое сообщество, 2019) · Руководство ASEAN по управлению ИИ и этике (многосторонний политический, 2024). Четыре институционально различимые формы — конвергенция является структурным свидетельством того, что принципиальные леса фреймворка не являются артефактом какой-либо одной традиции.
* **Структура**: 27 стабильных измерений (D01–D27, из `SEED_DIMENSIONS.yaml` v1.0; 16 подтверждены всеми четырьмя источниками, 11 — тремя). Каждый документ измерения содержит автоматически отрисованный нормативный раздел (цитаты по источникам, форма провода, заметки о конвергенции) и составленный человеком раздел реализации со ссылками `file:line`, которые должны проверяться grep-ом по текущей ветке main.
* **Дисциплина доказательств**: каждое числовое утверждение воспроизводимо из датированных, сгенерированных скриптами базовых линий (`compliance/baselines/`); проза может ссылаться на базовые линии, но никогда не встраивает непроверяемые числа. Четырёхуровневая иерархия валидации простирается от файлов реализации (источник истины) до публичных заявлений (`compliance/MEASUREMENT_METHODOLOGY.md`).
* **Дисциплина честности**: каждое измерение несёт инвентарь «Известных пробелов»; элементы, зависящие от субстрата, помечены именем владеющего субстрата, а не заявляются как выполненные. Десять сквозных выводов инвентаризированы в README соответствия (например, типизированные конверты провода ещё не эмитируются; семейство детекторов LensCore ещё не поставлено; пробел повторного рассмотрения обратной оси).

Приложение C остаётся в тексте как будущее место законодательных сопоставлений (статьи EU AI Act, NIST AI RMF, ISO/IEC 42001) после завершения юридического рецензирования; директория соответствия является **живой, доказательно насыщенной** таблицей соответствия до тех пор и питает её.

### 1.3 Формат провода: CEG заменяет FSD-002

Форматом провода федерации для линейки агентов CIRIS 3.0 является **CEG (CIRIS Epistemic Grammar)**, поддерживаемый в `CIRISRegistry/FSD/CEG/`. Грамматика заблокирована на **ровно пяти примитивах формата провода** — один рабочий (`scores`) плюс четыре структурных компоновщика (`delegates_to` / `supersedes` / `withdraws` / `recants`) — поверх открытого, описывающего механизмы пространства имён измерений. Блокировка удержалась на протяжении всей линейки 0.x; каждый инкремент представлял собой компоновку поверх уже заблокированных примитивов.

Для готовности к ASI важны два следствия:

1. **Поворот CEG-native**: агент 3.0 не *отображает* своё внутреннее состояние на формат провода — его мутации внутреннего состояния **являются** CEG-аттестациями на уровне себя под собственным ключом агента, продвигаемыми до видимости в Федерации согласно модели уровней. Доказательства, которые аудирует оператор, — это тот же конверт, который читает Федерация. Это реализованная форма требования Книги IX о том, чтобы поверхности ограничений были внешне ссылаемы, а не самоотчитываемы.
2. **Четыре реализации открывают CEG 1.0**: CIRISAgent (среда выполнения), CIRISNodeCore (консенсус), CIRISLensCore (обнаружение — внешний свидетель, которого агент никогда не эмитирует сам, анти-гудхартово разделение) и CIRISRegistry (авторитет). Ни одна сторона — включая самого агента — не может объявить верификационную ткань завершённой.

### 1.5 Уровень совести как критически важный инструмент управления в контексте ASI

Протокол CRE из Приложения D регулирует *пересечение порога*; уровень совести регулирует *каждое действие до и после него*. В реализованном виде (агент 2.9.6): каждая мысль проходит через стек проверок с приоритетной очерёдностью — четыре факультета, оцениваемые LLM (энтропия, когерентность, вето оптимизации, эпистемическая скромность), плюс детерминированный защитный барьер (ограничитель глубины мышления, проверка последовательности действий), который работает даже в benchmark-режиме и даже для действий, освобождённых иным образом от проверок. Пороговые значения являются константами уровня кода, а не обученными весами — это архитектурный инвариант, обеспечивающий невозможность устранения этики через многократное воздействие. Существует несколько независимых поверхностей остановки: отказобезопасное прерывание вето оптимизации, служба завершения работы и криптографически подписанный удалённый вызов Соглашения, инициирующий полную блокировку запрета. Каждая проверка записывает строку трассировки с тегом, поэтому аудитор может воспроизвести полную цепочку рассуждений для каждой мысли.

Известное ограничение, заявленное, а не скрытое: рефакторинг совести (3 фрагмента + детерминированный шлюз) частично выполнен; типизированная федеративная эмиссия вердиктов совести ожидает конверта Contribution субстрата. Подробности по каждому измерению: `compliance/D12_conscience.md`.

### 1.6 Траектория зависимости от субстрата

Верификация уровня ASI требует полной федеративной структуры. Траектория субстрата: **Persist → Edge → LensCore → NodeCore**; примерно треть из 27 измерений привязана к семейству внешних детекторов LensCore (свидетель, которого агент никогда не должен обеспечивать самостоятельно). До поставки этих субстратов соответствующие заявления по измерениям остаются *готовыми со стороны реализации, ожидающими со стороны федерации* — и помечены как таковые для каждого измерения.

### 1.7 Что это дополнение изменяет и что не изменяет

* Оно **не** повышает статус заявления ASI в Области применения — это заявление остаётся условным в отношении всех четырёх требований RC.
* Оно **не** заменяет Приложение D — протокол CRE остаётся неизменным в качестве шлюза пересечения порога.
* Оно **действительно** делает цепочку доказательств впервые полностью проверяемой: регуляторный абзац → измерение (D01–D27) → реализация `file:line` → датированная базовая линия → (по мере поставки субстратов) внешний федеративный свидетель.
* Оно **действительно** обязывает Соглашение обновлять это дополнение при каждой версии с текущим выпуском агента и базовой линией, чтобы заявление о готовности никогда не устарело незаметно.
